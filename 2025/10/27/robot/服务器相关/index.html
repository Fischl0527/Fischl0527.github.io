How to use Server

Created2025-10-27|Updated2025-11-15|robot
|Word Count:584|Reading Time:1mins|Post Views:

一、购买服务器

打开服务器的相关平台，选择合适的服务器进行购买

选择合适的参数进行购买

打开自己的容器，也就是开机

二、连接服务器

之后打开 vscode ，下载 remote ssh 插件

然后输入连接指令ssh -p 27272 root@connect.nmb1.seetacloud.com

其中登录指令直接在控制面版复制上就行

选择配置文件

然后点击箭头就可以连接了，操作系统选择 linux（因为我买的服务器是 linux）

输入密码 LRyrmN2heAQp

之后点击打开文件夹

此时会出现很多文件夹，autodl-pub，autodl-tmp。

选择合适的目录后点击确定，再次输入密码就可以打开文件夹了，我这里选择的是 root

autodl-pub目录里通常包含一些常用的公开数据集或模型资源。在开始你的项目前，可以先去这个目录看看是否有现成的数据可以利用，这能节省你的数据上传和准备时间

为了避免系统盘空间不足导致的问题，你的项目文件、代码、个人数据集以及训练得到的模型和日志，都应该放在 /root/autodl-tmp/ 目录下。

到此就算是完成了服务器的连接了

三、服务器上传文件

vscode 可以直接上传文件，直接拖文件到对应文件夹就行了，但是比较慢，不是很推荐

推荐 FileZilla 进行文件传输，下载好后打开程序点击新站点

输入相关信息即可，注意协议是 SFTP 协议

成功之后就是这样子的，也是直接拖文件进去就可以上传了

四、开始训练

先配好训练的相关环境，之后一定一定要挂后台进行训练，不然突然电脑断连训练就废掉了！推荐 tmux 命令

创建一个新的 tmux 会话：tmux new -s test

输入命令训练即可

相关命令：

tmux ls：查看所有会话

tmux attach -t test 进行连接

五、注意事项

（1）、不用的时候一定要关机！一定要关机！一定要关机！不然会一直扣钱！

（2）、如果 14 天不开机的话里面的文件自动清理
